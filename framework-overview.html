Project Overview

API Test Automation

A Spring Boot framework for scalable, annotation-driven API testing with automatic discovery, async execution, and transparent result persistence.

Framework Design
How the System Works
The framework uses annotation scanning, reflection, and AOP to discover, execute, and persist test results — with zero boilerplate inside test methods.
Full flow: POST /jobs → Controller → Service looks up definitions + saves TestPlanJobRecord (IN_PROGRESS) + fires @Async → returns 202 Accepted + jobId immediately → AsyncService loops tests via reflection → Aspect saves STARTED before each test runs → test executes → Aspect saves PASSED/FAILED with full details → client polls GET /test-case-results for live progress

POST /test-plans/{planName}/jobs Trigger a test plan run — returns 202 immediately
Request Body
{
  "description": "Pre-release smoke run"
}

Internal Flow
1
Controller validates planName exists in registry
2
Calls TestAutomationService
3
Service looks up List<TestCaseDefinition> — already sorted by suiteOrder → testCaseOrder
4
Saves TestPlanJobRecord with status = IN_PROGRESS
5
Fires TestOrchestrationAsyncService in background thread
6
Returns 202 Accepted with jobId — client begins polling
Response 202 Accepted
{
  "jobId": "f3a9c-uuid-...",

  "description": "Pre-release smoke run",
  "status": "IN_PROGRESS",

  "triggeredAtMs": 1708689600000
}
